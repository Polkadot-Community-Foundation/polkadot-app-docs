Architecture/The network

The network

The Polkadot Products Devnet is a public developer preview that runs on the community-operated Paseo network: a relay chain plus a set of system parachains. Applications are built against three of those chains — Asset Hub, People, and Bulletin.

!
Devnet. Tokens have no real value and flows may still change. Never paste secrets — mnemonics, seed phrases, private keys — into any tool or page.

Topology

The relay chain is the root of trust. Product flows mostly touch three system parachains: Asset Hub for contracts and assets, People for identity and money flows, and Bulletin for web-app bundle storage.

Asset Hub para 1000

Asset Hub is the primary chain for Product developers. It carries the contract, asset, and naming machinery used by deployed apps.

pallet-revive
The PolkaVM smart-contract environment. CDM, DotNS, and application contracts run here.
Assets + ERC-20 precompiles
On-chain assets exposed to contracts through ERC-20-style precompiles.
DotNS gateway
Connects .dot names to the contract environment.
Personhood precompile
Contracts read a user's personhood tier without calling the identity backend.

Bulletin para 1010

The Bulletin chain stores published web-app bundles that the gateway serves.

i
Bulletin uploads are authorization-based, not fee-based. Storage is gated by an authorizer rather than charged per upload — which is why publishing a bundle does not consume devnet tokens.

Deploy and serve flow

The three chains combine into a single application lifecycle: a developer publishes, the gateway resolves a .dot name to a bundle, and the user opens it.
